# 🌳 شجرة العائلة — دليل التطوير

## هيكل المشروع

| الملف | المسؤولية | متى تعدّله |
|-------|-----------|------------|
| `index.html` | هيكل HTML فقط | إضافة/إزالة عناصر واجهة |
| `styles.css` | التصميم والثيمات | تغيير الألوان أو التخطيط |
| `pwa.css` | تصميم بانرات PWA | تعديل بانر التثبيت |
| `js/pwa.js` | Service Worker + تثبيت PWA | مشاكل PWA |
| `js/mobile.js` | وضع الجوال + الثيم | مشاكل التصميم على الجوال |
| `js/core.js` | بيانات الأشخاص + فهارس | إضافة حقول جديدة للشخص |
| `js/search.js` | البحث + لوحة الجوال | تحسين البحث |
| `js/kinship.js` | حساب القرابة | تحسين خوارزمية القرابة |
| `js/export.js` | تصدير/استيراد | إضافة صيغة تصدير جديدة |
| `js/tree.js` | رسم الشجرة | تغيير طريقة عرض الشجرة |
| `js/cloud.js` | GitHub Gist | مشاكل الحفظ السحابي |
| `js/merge.js` | دمج تحديثات العائلة | تحسين آلية الدمج |
| `js/storage.js` | IndexedDB + FSA | مشاكل الحفظ المحلي |
| `js/supabase-sync.js` | مزامنة Supabase | مشاكل Supabase |
| `js/family-updates.js` | تحديثات العائلة | تحسين نظام التحديثات |

## إدارة النسخ بـ Git

```bash
# تهيئة Git (مرة واحدة فقط)
git init
git add .
git commit -m "v1.0 - البنية المنظمة الأولى"

# قبل أي تطوير جديد — أنشئ فرعاً آمناً
git checkout -b feature/اسم-الميزة

# بعد الانتهاء وإذا كانت الميزة تعمل
git add .
git commit -m "v1.1 - وصف ما أضفته"

# للرجوع لنسخة سابقة
git log --oneline          # اعرض قائمة النسخ
git checkout v1.0          # ارجع لنسخة معينة
git diff v1.0 v1.1         # اعرف ماذا تغير بين نسختين
```
